# Contributing to CAST-G

First off, thank you for considering contributing to CAST-G! It is people like you that make CAST-G such a great tool for the future of AI.

## 🚀 How Can I Contribute?

### Reporting Bugs
*   Use the GitHub Issue Tracker.
*   Describe the steps to reproduce the bug.
*   Include your hardware specs (CPU/GPU) and Python version.

### Suggesting Enhancements
*   Open an issue with the tag `enhancement`.
*   Provide a clear and concise description of the new feature.
*   If it is a research-based change, please include LaTeX formulas or paper references.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure your code follows the existing style (clean, modular, well-commented).
4.  Issue a pull request!

## 📜 Coding Standards
*   **Language**: Python 3.10+.
*   **Framework**: PyTorch (2.0+).
*   **Formatting**: We recommend using `black` for formatting.
*   **Documentation**: Every new module must include a Docstring explaining the math behind it.

## 🧪 Research Focus
CAST-G is a research-first project. We are particularly interested in:
*   New boundary detection algorithms.
*   Non-linear segment compression.
*   Hardware-aware attention mechanisms.
